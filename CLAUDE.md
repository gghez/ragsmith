# ragsmith

## Stack

- Python ≥ 3.14, fully async (`asyncpg`, `httpx`).
- Postgres + pgvector (raw SQL, no `pgvector` package).
- Voyage AI via raw HTTP (no `voyageai` SDK).
- `uv` for env, `ruff` for lint+format, `pytest` for tests.

## Hard rules

- **No `numpy` / `pandas`**, direct or transitive. Verify with `uv tree`.
- **100% coverage**, enforced (`--cov-fail-under=100`).
- **Ruff `select = ["ALL"]`**. Pre-commit runs `check` + `format --check`, no fix.
- **Google-style docstrings**.
- **Doctests run in CI** (`--doctest-modules`). Examples in docstrings must pass.
- **MkDocs strict build** must pass (`mkdocs build --strict`).
- **No silent bypasses.** Never add `# noqa`, `# type: ignore`, `# pragma: no cover`, per-file ignores, or relax any global rule (lint/coverage/strict) **without asking first**. Fix the code, don't suppress the rule. If suppression is truly needed, propose it with rule + reason and wait for go-ahead.

## Docs

- API ref auto-generated by `mkdocs-gen-files` (`scripts/gen_ref_pages.py`) + `mkdocs-literate-nav`. Don't hand-edit `docs/api/`.

## Release

- Bump `version` in `pyproject.toml` → tag `vX.Y.Z` → push tag.
- `release.yml` builds + publishes to PyPI via OIDC Trusted Publisher + creates GitHub Release.

## Local dev

- `docker compose up -d` for Postgres + pgvector.
- Secrets in `.env` (gitignored), template in `.env.example`.
